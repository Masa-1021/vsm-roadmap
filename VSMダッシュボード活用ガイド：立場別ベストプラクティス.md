# VSMダッシュボード活用ガイド：立場別ベストプラクティス

本ガイドは、VSMダッシュボード（Phase 1）を**管理者・現場・生産技術**の3つの立場でどう活用すれば最大の効果を得られるかを整理したものである。

---

## 1. 管理者（Management）向け

### このアプリで何ができるか

管理者モードでは、工場全体の「健康状態」を一目で把握し、**損失を金額で見て、最も効果的な手の打ち方を判断する**ことができる。

### 使える画面と機能

| 画面 | 何がわかるか |
|------|-------------|
| 全ライン健康状態一覧 | 全ラインの稼働/注意/停止を俯瞰。各ラインのスループット・直行率・仕掛品・損失金額 |
| 工場全体ライン健康状態一覧（L1） | 総生産量vs計画、計画達成率、工場全体OEE、ライン別ボトルネック状況 |
| VSMダッシュボード（管理者モード） | 特定ラインのVSMフロー上で各工程の推定損失・遅延リスクをリアルタイム表示。AIサマリーで優先度順のアラート |
| 日次稼働実績レポート | 生産実績/計画比、直行率、損失金額、パレート分析（損失要因の内訳）、イベントタイムライン |

### ベストプラクティス：1日の使い方

#### 朝（始業時）：「今日、どこが危ないか」を30秒で掴む

**全ライン健康状態一覧** を開く。

- 赤（停止）のラインがあれば、その損失金額を確認する
  - 例：Line A 停止中 → 損失 ¥1,250,000 → 最優先で復旧指示
- 黄（注意）のラインは直行率を確認する
  - 例：Line E 直行率78.5% → 品質問題の兆候 → 生産技術に調査依頼

**判断基準：損失金額の大きいラインから対応する。** 「停止しているから」ではなく「最もお金を失っているから」手を打つ。

#### 午前中：「着地できるか」をリアルタイムで監視する

**VSMダッシュボード（管理者モード）** を確認する。

- 画面下部の **生産進捗バー**（例：目標1,000に対して850、達成率85%）で着地予測を見る
- 各工程ノードの **遅延リスク**（例：High 85%）が高い箇所を特定する
- 右サイドバーの **AIサマリー** を確認する。優先度「高」のアラートから対処する
  - 例：「推定損失 ¥50,000/日」→ 放置すると月間150万円の損失

**判断基準：計画乖離5%のアラートが出たら、その時点で応援投入・ライン再編を判断する。** 16時まで待たない。

#### 午後：「残業が必要か」を早期に判断する

進捗バーとAI着地予測を再確認する。

| 状況 | 判断 |
|------|------|
| 達成率90%以上、遅延リスクLow | 定時で完了。追加対応不要 |
| 達成率80-90%、遅延リスクMedium | 応援を投入すれば定時完了の可能性。隣接ラインの余力を確認 |
| 達成率80%未満、遅延リスクHigh | 残業を承認するか、翌日に計画を振り替えるか判断 |

**利益への効果：** 残業判断を終業2時間前に行えるようになる。無駄な残業の削減と、必要な残業の早期開始（効率的）が可能。

#### 終業時～週次：「どこに投資すれば利益が戻るか」を見極める

**日次稼働実績レポート** の **損失要因分析（パレート）** を確認する。

- 例：チョコ停 42.3%（¥52,600）、段取り替え 25.1%（¥31,250）、品質不具合 18.6%（¥23,150）
- **AI要因インサイト** が具体的な原因を提示する
  - 例：「チョコ停の主因はコンベア搬送部の位置ずれ。14時台に集中、周辺温度上昇が影響」

**判断基準：改善投資のROIを即座に計算する。**
- センサー交換費用 ¥200,000 ÷ 日次損失 ¥52,600 = **3.8日で投資回収**
- この数字を持って、改善の起案・承認を行う

### 管理者が犯しがちな間違い

- **「緑のラインは見なくていい」と思うこと** → 直行率が微妙に下がり始めていないか、仕掛品が増え始めていないかを見逃す
- **AIアラートを全部同じ重みで見ること** → 優先度「高」の損失金額を見て、金額の大きい順に対応する
- **月次でまとめて見ること** → 毎日の着地管理を行い、問題を小さいうちに潰す

## 2. 現場（Operation）向け

### このアプリで何ができるか

現場モードでは、**自分の担当ラインが「今、計画通りか」を一目で確認し、異常があれば即座に行動に移す**ことができる。

### 使える画面と機能

| 画面 | 何がわかるか |
|------|-------------|
| VSMダッシュボード（現場モード） | 各工程の目標タクト vs 実績タクト、稼働率、仕掛品（WIP）の過不足、異常停止状況 |
| アラート・ボトルネック強調画面 | ボトルネック工程をハイライト表示。バッファ滞留の警告。アラート通知（部材不足、段取り替え予告など） |
| 生産進捗バー | 本日の生産数 vs 目標、進捗率、ペース判定（「+12個 ペース良」等） |

### ベストプラクティス：1日の使い方

#### 朝（始業時）：「今日のゴール」を確認する

画面下部の **生産進捗バー** を見る。

- 例：目標800個、現在0個 → 今日のゴールが明確
- 目標タクト（例：60s）を確認し、このペースで回せば何時に終わるかを頭に入れる

#### 作業中：「3つの数字」だけ見ればいい

現場モードで常に監視すべきは以下の3つだけ。

| 何を見る | どう判断する |
|---------|-------------|
| **実績タクト vs 目標タクト** | 目標60sに対して実績58s → 順調。実績80s → 遅れている。原因を確認 |
| **仕掛品（WIP）の色** | 青（適正）→ OK。赤（在庫過多）→ 後工程が止まっている。状況確認。空（在庫ゼロ）→ 前工程が遅れている |
| **工程ステータスの色** | 緑（稼働中）→ OK。黄（段取替中）→ 準備作業に入る。赤（異常停止）→ アンドンを呼ぶか自分で対応 |

#### アラートが来たら：「見て→判断→行動」を1分以内に

右サイドバーの **マイアラート** は、自分に関係する通知だけが表示される。

- **赤（HIGH PRIORITY）：即座に行動**
  - 例：「段取り替えまであと10分。治具Aを準備してください」→ 工具を取りに行く
  - 例：「工程3 センサー異常予測」→ 保全に連絡 or 自分で点検
- **黄（WARNING）：5分以内に確認**
  - 例：「部品トレイBの残量低下。15分以内に不足」→ 補充要請 or 自分で補充
- **青（INFO）：把握のみ**
  - 例：「シフト交代予定」「休憩時間変更」

#### 帰る前：「あと何個」で達成感を得る

進捗バーを確認し、目標達成したかを確認する。

- 達成 → そのまま引き継ぎ
- 未達 → 次のシフトへ残件数を共有（VSMが自動で引き継ぎ情報を生成する）

### 現場で最も効果的な使い方

- **ボトルネック強調画面を使って「詰まり」を見つける**
  - バッファに赤い数字（例：65/Max30）が出ていたら、後工程が止まっている証拠。自分の工程を止めて応援に行くべきか、リーダーに相談する
- **「アンドン呼び出し」ボタンを躊躇なく使う**
  - 異常を1分放置すると後工程が止まる。アプリの「アンドン呼び出し」ボタンから即座にヘルプを要請する
- **タクトの「体感」とデータを照合する**
  - 「なんか今日遅いな」と感じたら画面を確認する。実績タクトが目標を超えていれば感覚は正しい。原因を特定して対処する

### 現場が犯しがちな間違い

- **アラートを「後で見る」こと** → アラートは「今すぐ」行動するためのもの。後で見るなら意味がない
- **仕掛品の量を無視すること** → 自工程だけ速くても、後工程が止まっていたら仕掛品が溜まるだけ。全体の流れを見る
- **異常時に「自分で何とかしよう」と時間をかけること** → 3分以内に解決しなければアンドンを呼ぶ。それが全体のためになる

## 3. 生産技術・工技（Engineering）向け

### このアプリで何ができるか

生産技術モードでは、**設備と工程のパフォーマンスをデータで深掘りし、改善のための根拠を得る**ことができる。

### 使える画面と機能

| 画面 | 何がわかるか |
|------|-------------|
| 工程詳細分析・AIレポート | 特定工程の損失コスト推移（30日）、AI分析レポート（原因特定）、スキルマップ（担当者の能力評価）、SOP動画 |
| 機種別初品検査結果ダッシュボード | 検査ステータス一覧、詳細項目チェック（寸法・外観・機能）、測定データグラフ（公差内のばらつき）、承認フロー |
| 日次稼働実績レポート（共用） | 損失要因パレート分析、イベントタイムライン、AIインサイト |

### ベストプラクティス：主要な業務シーン別

#### シーン1：設備トラブルの根本原因を突き止める

**工程詳細分析・AIレポート** を開く。

1. **AI分析レポート** を最初に読む
   - 例：「リードタイム延長の主原因：頻繁なチョコ停（平均15分間隔）と特定シフトにおける作業者のスキル不足との相関が高い。午後2時〜4時に効率低下が顕著」
2. **損失コスト推移（30日グラフ）** で、問題がいつから始まったかを特定する
   - 急増ポイントがあれば、その前後に何が変わったか（部品ロット変更、人員交代、気温変化等）を調査する
3. **スキルマップ** で、人の問題かどうかを確認する
   - 例：鈴木一郎（新人）→ 組立速度「未熟」、正確性「要指導」、保守「未熟」→ 評価C
   - この人が担当する時間帯と効率低下の時間帯が一致すれば、教育の問題
4. **SOP動画** のAI提案を確認する
   - 例：「新人作業者に対して、2分15秒からの『コネクタ接合時の注意点』を重点的に教育することを推奨」

**行動：** 設備の問題か人の問題かを切り分け、適切な対策（修理/教育/SOP改訂）を実行する。

#### シーン2：初品検査で品質を確保する

**機種別初品検査結果ダッシュボード** を使う。

1. **検査ステータス一覧** で全体の進捗を確認する
   - 例：本日12機種中、合格10、不合格1、保留1
2. 不合格品を選択し、**詳細項目チェック** を確認する
   - 例：Galaxy S24 Frame → 「表面傷・打痕チェック」が不合格
3. **検査ポイント写真** で実際の状態を目視確認する
4. **測定データグラフ** で公差内のばらつきを確認する
   - 中心線に対してどちら方向にずれているか → 金型の摩耗方向や温度影響を推測する
   - UCL/LCLに近づいているポイントがあれば、量産前に調整が必要

**行動：** 不合格の根本原因を特定し、量産開始前に対策を完了する。承認フローで管理者に報告する。

#### シーン3：標準時間（ST）の精度を上げる

**日次稼働実績レポート** の実績データを活用する。

1. 計画上の標準時間と、VSMが計測した実績タクトを比較する
   - 例：ST=60sに設定しているが、実績平均は65s → STが甘い（現場に無理を強いている）
2. 品種ごとの実績タクトのばらつきを確認する
   - 特定品種だけ遅い場合 → その品種固有の作業難易度が反映されていない
3. 損失要因パレートの「段取り替え」時間を確認する
   - 計画上の段取り時間と実績に差があれば、次回の計画精度を修正する

**行動：** 実績データに基づいてSTを更新し、計画精度を向上させる。これにより管理者の着地予測精度も上がる。

#### シーン4：改善効果を定量的に証明する

改善施策の実施前後で、VSMの以下のデータを比較する。

| 指標 | 改善前 | 改善後 | 効果 |
|------|--------|--------|------|
| 対象工程のサイクルタイム | 80s | 62s | -22.5% |
| チョコ停回数/日 | 15回 | 3回 | -80% |
| 推定損失金額/日 | ¥52,600 | ¥8,000 | -84.8% |
| 直行率 | 89.2% | 97.5% | +8.3pt |

**行動：** この比較データを改善報告書に使い、次の改善投資の承認を得る材料にする。

### 生産技術が犯しがちな間違い

- **AI分析レポートを鵜呑みにすること** → AIは相関を示すが因果は示さない。現場に行って裏を取ることが必須
- **測定データの「公差内」で安心すること** → 公差内でも中心からの偏りが大きければ、いずれ公差を超える。Cpk（工程能力指数）の観点で判断する
- **改善効果を「体感」で報告すること** → VSMが自動で前後比較データを出してくれる。数字で報告することで、次の投資承認が格段に通りやすくなる

## まとめ：3つの立場の連携が利益を生む

VSMダッシュボードの真の価値は、3つの立場が**同じデータ**を見て、**同じ言語（数字）**で会話できることにある。

```
現場が異常を検知 → アラートで管理者・生産技術に共有
  ↓
生産技術がAIレポートで根本原因を特定 → 改善策を立案
  ↓
管理者が損失金額とROIで投資判断 → 改善を承認
  ↓
改善実施 → VSMで効果を自動検証
  ↓
次の改善サイクルへ
```

この**データ駆動の改善ループ**が回り始めれば、VSMは「見える化ツール」から「利益を生む経営基盤」に変わる。
