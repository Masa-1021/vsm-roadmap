# VSM進化ロードマップ：プロジェクトの進め方

> **現場の「見える化」を、3段階で進化させる。**
> 紙のVSMから、リアルタイムVSM、そしてデジタルツインへ。

---

## 全体像：3つのPhaseと進化の流れ

```mermaid

    subgraph Phase1["Phase 1：予実同期と異常の見える化"]
        P1A[計画 vs 実績の<br>リアルタイム表示]
```

### Phaseごとの位置づけ

| | Phase 1 | Phase 2 | Phase 3 |
|---|---|---|---|
| **テーマ** | 予実同期と異常の見える化 | AIによる予測管理とリソース最適化 | デジタルツインによる自律経営 |
| **一言で** | 「今どうなっている？」が見える | 「次に何が起きる？」がわかる | 「どうすれば最善か？」を試せる |
| **核心技術** | リアルタイムデータ収集・可視化 | AI分析・予測モデル | シミュレーション・最適化エンジン |
| **ダッシュボード数** | 8画面 | 11画面 | 4画面 |
| **状態** | **作成中** | 設計済 | 設計済 |

## Phase 1：予実同期と異常の見える化

### 目指す姿

> **「今、計画に対してどうなのか」を全員がリアルタイムに共有し、即応できる体制を作る**

```mermaid
flowchart LR
    PLAN[生産計画] --> COMPARE{予実比較}
    ACTUAL[リアルタイム実績] --> COMPARE
    COMPARE -->|乖離 < 5%| OK[正常運転<br>緑表示]
    COMPARE -->|乖離 ≥ 5%| ALERT[アラート発報<br>赤表示]
    ALERT --> ACTION[即時対応<br>応援・調整]
    ALERT --> LOSS[損失金額<br>自動算出]

    style ALERT fill:#ffcdd2,stroke:#c62828
    style OK fill:#c8e6c9,stroke:#2e7d32
    style LOSS fill:#fff9c4,stroke:#f57f17
```

### Phase 1 で得られる価値

| 対象 | 何が変わるか | 具体例 |
|------|-------------|--------|
| **管理者** | 計画達成率をリアルタイム予測、損失を金額で把握 | 「Line A 停止中 → 損失 ¥1,250,000」で優先度を即判断 |
| **現場** | 目標タクト vs 現在タクトで自律的に調整 | 「あと何個」のカウントダウンで達成感 |
| **生産技術** | 計画外停止のインパクトを自動算出 | 標準時間(ST)を実績データで自動校正 |
| **共通** | 計画乖離5%でプッシュ通知 | 問題が大きくなる前に手を打てる |

## Phase 2：AIによる予測管理とリソース最適化

### 目指す姿

> **「ヒト・モノ・設備」の状態をAIが解析し、先回りして無駄を排除する**

```mermaid
flowchart TD
    DATA[Phase 1で蓄積された<br>リアルタイムデータ] --> AI["AI分析エンジン"]

    AI --> PREDICT["予測<br>「次に何が起きるか」"]
    AI --> DETECT["検知<br>「微小停止の共通原因」"]
    AI --> OPTIMIZE["最適化<br>「人員・段取りの最適配置」"]

    PREDICT --> ALERT2["先回りアラート<br>「15分後に部品不足」"]
    DETECT --> ROOT["根本原因の特定<br>「センサ汚れが主因」"]
    OPTIMIZE --> AUTO["自動化<br>「日報自動生成」"]

    style AI fill:#e3f2fd,stroke:#1565c0,stroke-width:3px
    style ALERT2 fill:#ffcdd2,stroke:#c62828
    style AUTO fill:#c8e6c9,stroke:#2e7d32
```

### Phase 2 で得られる価値

| 対象 | 何が変わるか | 具体例 |
|------|-------------|--------|
| **管理者** | スキルマトリックスと進捗の掛け合わせで高精度な着地予測 | ライン間の応援タイミングを最適指示 |
| **現場** | 「次の一手」をウェアラブルで通知、日報が自動生成 | 段取り替えカウントダウンで準備を最適化 |
| **生産技術** | 微小停止のクラスタリング、品種別ボトルネック特定 | 段取り時間の品種組み合わせ影響を分析 |

## Phase 3：デジタルツインによる自律経営

### 目指す姿

> **「シミュレーション」が現場を動かし、未来の利益を最大化する**

```mermaid
flowchart LR
    REAL["現実の工場<br>（フィジカル）"] <-->|双方向同期| TWIN["デジタルツイン<br>（サイバー）"]

    TWIN --> SIM1["What-If シミュレーション<br>「特急品を入れたらどうなる？」"]
    TWIN --> SIM2["投資シミュレーション<br>「設備導入の効果は？」"]
    TWIN --> SIM3["プロダクトミックス最適化<br>「最大利益の品種構成は？」"]

    SIM1 --> DECIDE["経営判断"]
    SIM2 --> DECIDE
    SIM3 --> DECIDE
    DECIDE -->|最適パラメータを送信| REAL

    style REAL fill:#e8f5e9,stroke:#2e7d32,stroke-width:2px
    style TWIN fill:#e3f2fd,stroke:#1565c0,stroke-width:2px
    style DECIDE fill:#fff3e0,stroke:#e65100,stroke-width:2px
```

### Phase 3 で得られる価値

| 対象 | 何が変わるか | 具体例 |
|------|-------------|--------|
| **管理者** | 投資判断をシミュレーションで事前検証、リアルタイムPL可視化 | 利益率の高い品種構成を自動探索 |
| **現場** | 作業標準が品種に合わせて自動切り替え | バイタルデータで安全管理（疲労度検知） |
| **生産技術** | 特急品投入の影響をシミュレーション、設備パラメータの自動最適化 | 治具・工具の寿命を予兆管理 |

## データ駆動の改善ループ

3つのPhaseを通じて構築される改善サイクル：

```mermaid
graph TD
    A["1. 現場が異常を検知<br>（Phase 1：リアルタイム監視）"] --> B["2. AIが根本原因を特定<br>（Phase 2：予測分析）"]
    B --> C["3. シミュレーションで<br>改善策を事前検証<br>（Phase 3：デジタルツイン）"]

    E --> F["6. VSMで効果を<br>自動検証"]
    F --> A
```
